Se realizó una centlización de las dolencias y enfermddes que pueden presentar los pacientes, junto con el nivel de prioridad asociado a cada una.

para mantener una lógica consistente en todo el sistema y evitar reglas dispersas en el código

El sistema no solo se basa en dolencias declaradas, sino también en la detección automática de anomalías clínicas a partir de los signos vitales.

Se contemplan casos como:

Problemas respiratorios
Taquicardias
Arritmias
Fiebre
Hipertensión
Baja saturación de oxígeno
etc

Estas condiciones se identifican mediante umbrales definidos en el código, los cuales permiten interpretar los signos vitales del paciente y detectar situaciones críticas.
Los pacientes proporcionados en el taller original fueron transformados y adaptados a este nuevo formato estructurado, permitiendo su procesamiento dentro del sistema de priorización.
